# AI 산불 조기감시 관제 콘솔

제1회 산림과학 AI 활용 경진대회 제출용 Prototype Dashboard.
공공 산불·기상 데이터를 AI로 분석해 고위험 알림을 관제 우선검토 큐에 배치하는 감시 웹사이트다.

※ 본 화면은 2024~2025년 holdout 데이터를 재생한 데모이며, 실제 산불 여부는 사후검증 라벨이다.

배포 URL: https://leeminhong.github.io/ai-wildfire-monitoring-dashboard/
QR 코드: `assets/pages-qr.png`

## 최신 승인 수치

- test universe: **30,788건**, 사후검증 산불 **384건**
- 대표 운영점: 위험 하위 **40% 자동검증 후보**, 상위 **60% 관제 우선검토**
- 자동검증 후보 **12,315건** / 관제 우선검토 **18,473건**
- 후순위 분류 산불 **29건** / 관제큐 산불 포착률 **92.4%**
- 상위 20% 우선검토 큐 산불 집중도 **2.318x**
- ROC-AUC **0.736** (95% CI 0.7152-0.7572)

## 화면에서 보여주는 정보

- 위치: 시도, 시군구, 날짜별 감시 우선순위
- AI 위험점수: 모델 출력값을 0-100 스케일로 표시하되, 보정된 실제 발생확률이 아니라 관제 우선순위 점수로 해석
- 주요 위험요인: 최고기온, 평균습도, 최대풍속, 강수량
- 정적 취약성: KOSIS 농지면적, 미매칭 11개 시군구는 train median 모델입력 대치로 표시
- 상태: 우선검토, 주의, 사후검증: 실제 산불
- 데이터 품질 로그: 농지 미매칭 11개 시군구를 대시보드에 투명하게 표시

## 구성

- `index.html` - 단일 파일 관제 콘솔. 빌드 없이 브라우저에서 바로 열 수 있다.
- `assets/figures/` - 검토량-놓침 곡선, Top-K 집중도, ROC 등 제출 근거 그림 보관용.
- `.nojekyll` - GitHub Pages Jekyll 비활성화.

## 실행 방식

정적 사이트라 별도 빌드가 필요 없다.

```bash
python3 -m http.server 8777 --bind 127.0.0.1
```

브라우저에서 `http://localhost:8777/index.html`을 열면 바로 대시보드가 표시된다. GitHub Pages에 배포할 때는 이 폴더의 `index.html`이 저장소 루트에 오면 된다.

## 데이터·도구 출처

- 산불통계: data.go.kr 15121380 (산림청)
- 산악기상: data.go.kr 15067450 / apis.data.go.kr 1400377 계열 (국립산림과학원)
- 기상 backbone: 기상청 ASOS
- 정적 feature: KOSIS 시군별 논밭별 경지면적
- 모델: scikit-learn Logistic Regression / train-only SMOTE
- 검증 파이프라인: Codex/OpenClaw 멀티에이전트 handoff

## 제출 체크리스트

- GitHub Pages 링크 접속 시 `index.html` 대시보드가 바로 표시되어야 한다.
- 웹사이트 숫자와 제안서 숫자는 `30,788 / 12,315 / 18,473 / 29 / 92.4% / 2.318x / 0.736`으로 일치해야 한다.
- 발표·제안서에서는 `실제 산불`을 운영시점 정보가 아니라 `사후검증 라벨`로 설명한다.
- PC 기준 레이아웃을 우선 검토한다.
- QR 코드 `assets/pages-qr.png`를 제안서 5번 서비스 구현 화면에 삽입한다.
