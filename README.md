# 산불 위험 조기탐지 자율검증 시스템 — 대시보드

제1회 산림과학 AI 활용 경진대회(국립산림과학원) **AI기술활용상** 출품작의 서비스 대시보드.
공개 산불통계 + 국립산림과학원 산악기상 데이터로 시군구·일 단위 산불 위험을 예측하고,
AI가 판단을 스스로 검증해 관제 검토시간을 줄이는 **자율검증 triage** 결과를 시각화한다.

## 핵심 결과
- 자동검증율 **40%** → 관제 검토 23,404 → 14,042건 (**검토시간 40% 단축**)
- 우선큐 산불 집중도 **2.50×**, 발화 전일 포착률 **71.8%**
- 놓친 산불 **24건**(7.95%, 95% CI 4.97–11.26%) · 기상결측 예측불가 **82건** — 비용으로 공개
- 재현 ROC-AUC **0.768**(논문 0.64–0.81 범위 내), Recall **0.639**(논문 0.718 미달, 정직 보고)

## 구성
- `index.html` — 단일 파일 대시보드(외부 의존성: Pretendard 폰트 CDN만). 빌드 불필요.
- `assets/figures/` — 멀티에이전트 파이프라인 산출 분석 그림(PNG/SVG).
- `.nojekyll` — GitHub Pages Jekyll 비활성화.

## 디자인 원칙
에디토리얼 미니멀 · 3색 토큰(paper `#F4F5F1` / ink `#15201A` / ember `#D8472A`, 60-30-10) ·
8px 스페이싱 · 12컬럼 그리드 · 단일 포컬포인트(40%) · soft shadow · 200–300ms ease-out 인터랙션 ·
버튼/링크 4상태(hover·focus·active·disabled). 모든 토큰은 `:root` CSS 변수.

## GitHub Pages 배포
1. 새 저장소 생성 후 이 폴더 내용을 push (`index.html`이 루트에 있어야 함).
2. Settings → Pages → Source: `main` 브랜치 `/ (root)`.
3. 게시 후 `index.html`의 GitHub 버튼 `href`(주석 `repoBtn`)를 저장소 URL로 교체.
